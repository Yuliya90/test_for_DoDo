# Прототип системы детекции уборки столиков по видео

## Описание:

Скрипт анализирует видеозапись и отслеживает три состояния выбранного столика:
- **EMPTY** — столик пуст (людей в зоне нет)
- **OCCUPIED** — столик занят (человек в зоне)
- **APPROACH** — подход к столику (появление человека после периода пустоты)

На основе событий считается **среднее время между уходом гостя и подходом следующего человека**.

---

## Установка зависимостей

```bash
pip install -r requirements.txt
```
При первом запуске YOLOv8n скачается автоматически.

---

## Запуск

```bash
python main.py --video "видео 1.mp4"
```

### Параметры

Параметры
`--video` Путь к входному видео (обязательно) 
`--output` | `output.mp4`  Путь к выходному видео с визуализацией 
`--report` | `report.txt` Путь к текстовому отчёту 
`--model` | `yolov8n.pt` Путь к модели YOLO 
`--roi X Y W H` Координаты ROI

### Пример с ручным заданием ROI

```bash
python main.py --video видео 1.mp4 --roi 320 180 200 150
```

---

## Как выбрать столик (ROI)

При запуске без `--roi` откроется окно с первым кадром видео.
Выделите зону столика мышью и нажмите **ENTER** для подтверждения.

---

## Логика детекции событий

1. **Модель**: YOLOv8n (Ultralytics) — обнаруживает людей (класс `person`) на каждом кадре.
2. **Пересечение с ROI**: если bounding box человека перекрывает зону столика более чем на 15% площади ROI — человек считается "в зоне".
3. **Debounce**: состояние меняется только если оно стабильно на протяжении 15 кадров подряд — это устраняет ложные срабатывания.
4. **Событие APPROACH**: фиксируется при переходе `EMPTY → OCCUPIED`.
